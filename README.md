# 🌐 HTTP vs HTTPS

## 📌 Overview
This repository explains the differences between **HTTP (HyperText Transfer Protocol)** and **HTTPS (HyperText Transfer Protocol Secure)** — the core protocols used for communication on the web.

Understanding these protocols is essential for building secure and reliable web applications.

---

## 🔍 What is HTTP?

**HTTP** is a protocol used for transferring data between a client (browser) and a server.

### ⚙️ Features:
- Works on port **80**
- Fast and lightweight
- Data is transferred in **plain text**
- No encryption or security

### ⚠️ Limitations:
- Vulnerable to attacks (e.g., Man-in-the-Middle)
- No data protection
- No authentication

---

## 🔐 What is HTTPS?

**HTTPS** is the secure version of HTTP that uses **SSL/TLS encryption** to protect data.

### ⚙️ Features:
- Works on port **443**
- Data is **encrypted**
- Ensures **secure communication**
- Provides authentication using SSL/TLS certificates

### ✅ Advantages:
- Protects sensitive data (passwords, credit cards)
- Prevents data tampering
- Builds user trust (🔒 padlock in browser)

---

## 🧠 Why HTTPS is Important

- Protects user data from hackers  
- Ensures privacy and confidentiality  
- Improves website credibility  
- Required for modern web features (e.g., payments, login systems)

---

## 🛠️ How HTTPS Works (Simple Flow)

1. Client requests secure connection  
2. Server sends SSL/TLS certificate  
3. Client verifies certificate  
4. Encrypted connection is established  
5. Secure data transfer begins.
